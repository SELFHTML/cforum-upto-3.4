Classic Forum

Introduction

The Classic Forum is a web based forum project designed for high performance, scalability and customization. It's a classic thread based forum implemented in C and Perl. It has been successfully tested with FreeBSD and Linux but should also run on other Unizes. It has been written from scratch for the german SELFHTML Project and is productive since version 1.0 which came out last quartal 2002.

Under The Hood

The Classic Forum follows a client-server based concept using UNIX sockets for fast communication between the server and the client. It optionally provides the use of shared memory to accellerate the communication. The server program has the whole data in memory, so there is no need for the client software to parse the data again and again. In a defined interval the data will be written to disk using XML for preventing loss of data. Of course the client and the server are running on the same machine, there is no need for the user to install additionally software. He only needs a web browser. It also has an easy-to-use plugin API which makes it easy to implement new features.

Status Of Developement

The Classic Forum is in productive use since the the 11th November 2002. The most recent version is 2.0 Beta 1. The most recent stable version is 1.2. Both versions are available at the download section of this page. The recent version implements the following features:

  • Basic Forum routines like posting, reading threads, etc.
  • User-specific display options of the forum:
    • Defining your own font settings (e.g. family, size and color)
    • Using a frameset to view posting and main page simultanously
    • Collapsing of threads (with and without JavaScript)
    • Marking of threads/postings by posters or categories
    • Filter thread list live by different criterias
    • Highlight your own threads/postings
    • User defined and global ignore lists and VIP lists
    • User defined quoting characters
    • Highlighting of already visited postings
    • Automatic reload of the thread list
    • The use of an user-defined CSS file
    • etc, etc, etc

special features:

  • An implementation of the NNTP protocol to the forum
  • Correct HTTP/1.1 If-Modified-Since handling and caching headers to avoid traffic and load
  • The forums own macro language to create very dynamic filters
